应用框架 ​

应用框架是 优享云盘 云存储应用的主要协调组件。它提供了主要的布局结构、导航系统、用户管理界面，并协调应用各主要功能区域之间的通信。

架构概述 ​

Framework 组件实现了经典的应用外壳模式，提供了一个固定的头部、侧边栏导航和动态内容区域。它管理全局应用状态，包括用户认证、菜单导航和空间使用跟踪。

组件层次结构：

  • Framework.vue（主外壳）
  • Uploader.vue（文件上传界面）
  • UpdateAvatar.vue（头像管理）
  • UpdatePassword.vue（密码管理）
  • 通过 router-view 加载的动态路由组件

布局结构 ​

Framework 实现了一个三部分的布局：头部、侧边栏和主内容区域。

头部组件 ​

头部包含应用品牌标识、上传面板入口和用户账户控件。

关键元素：

  • Logo：显示带有图标和文本的 优享云盘 品牌标识
  • 上传面板入口：点击图标按钮可打开上传弹出窗口
  • 用户下拉菜单：显示用户头像、昵称和账户选项

侧边栏导航 ​

侧边栏提供两级导航，包含主菜单部分和二级菜单项。

菜单系统数据结构： 菜单配置定义在 menus 数组中

包含：

  • 主菜单：按类别浏览文件（所有、视频、音乐、图像、文档、其他）
  • 分享菜单：文件分享管理
  • 回收站菜单：恢复已删除的文件
  • 设置菜单：仅管理员可见的系统配置

菜单系统实现 ​

菜单系统使用分层数据结构，并根据用户权限进行计算过滤。

菜单配置 ​

菜单属性：

  • menuCode：用于路由匹配的唯一标识符
  • path：导航的路由路径
  • icon：用于显示的图标类名
  • name：显示名称
  • allShow：可见性标志（所有用户可见为 true）
  • children：子菜单项数组

权限过滤：filteredMenus 计算属性根据用户的管理员状态过滤菜单可见性。

导航逻辑 ​

菜单导航通过 jump 方法和路由监听处理：

导航流程：

  1. 用户点击菜单项 → 调用 jump()
  2. 路由器导航到新路由
  3. 路由监听器检测到变化
  4. setMenu() 更新当前菜单状态

用户管理集成 ​

Framework 管理用户认证状态并提供账户管理界面。

用户状态管理 ​

用户数据结构：

  • userInfo：包含从 cookie 中获取的用户详细信息
  • timestamp：用于使头像缓存失效

账户管理操作：

  • 更新头像：触发 UpdateAvatar 组件
  • 更改密码：触发 UpdatePassword 组件
  • 注销：确认并处理注销操作

文件上传集成 ​

Framework 通过事件处理和组件通信与文件上传系统紧密集成。

上传工作流程 ​

关键部分

  1. 添加文件：子组件触发 addFile 事件
  2. 触发上传：Framework 显示上传器并传递文件数据
  3. 完成回调：更新文件列表和空间使用情况

组件引用：

  • uploaderRef：对 Uploader 组件的引用
  • routerViewRef：对当前路由组件的引用

空间使用跟踪 ​

Framework 实时显示和管理用户存储信息，并进行更新。

空间使用实现 ​

空间管理功能：

  • 实时显示：通过进度条显示已使用空间与总空间的比例
  • 手动刷新：带有旋转动画的防抖刷新按钮
  • 自动更新：文件操作后自动刷新

API 集成：

  • 端点：/getUseSpace
  • 响应结构：{useSpace, totalSpace}

防抖实现： 自定义防抖函数可防止过多的 API 调用

路由集成 ​

Framework 作为路由出口容器，实现动态组件渲染和事件处理。

路由视图实现 ​

动态组件特性

  • 作用域插槽：访问组件实例以进行事件绑定
  • 事件绑定：监听子组件的 addFile 事件
  • 引用管理：维护 routerViewRef 以直接调用方法

路由监控： Framework 监听路由变化，根据路由元数据更新菜单状态

API 通信 ​

Framework 通过有限的端点与后端服务进行通信，以实现核心功能。

API 端点 ​

端点用途使用场景
/logout用户注销认证管理
/getUseSpace空间使用情况存储跟踪

API 调用模式： 使用全局 proxy.Request 方法进行一致的 HTTP 处理
