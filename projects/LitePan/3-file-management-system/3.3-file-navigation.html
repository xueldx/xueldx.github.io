// 在 Navigation.vue 中，openFolder 方法更新导航路径
const openFolder = (data) => {

  folderList.value.push(folder)
  currentFolder.value = folder

}

导航控制 ​

导航组件提供了几种导航方法：

方法用途实现细节
openFolder(data)导航到子文件夹将文件夹添加到路径并更新当前文件夹
setCurrentFolder(index)导航到路径中的特定位置在给定索引处截断路径，如果索引为 -1 则清除路径
backParent()返回父文件夹使用父文件夹的索引调用 setCurrentFolder
setPath()更新 URL 以反映当前路径从文件夹 ID 构建查询参数

URL 集成 ​

导航组件与 URL 同步，以支持直接链接到特定文件夹：

javascript
// 更新 URL 以匹配当前导航路径
const setPath = () => {
  if (!props.watchPath) {
    doCallback()
    return
  }
  const pathArray = []
  folderList.value.forEach((item) => {
    pathArray.push(item.fileId)
  })
  router.push({
    path: route.path,
    query: pathArray.length == 0 ? '' : { path: pathArray.join('/') },
  })
}

该组件还会监听 URL 的变化，并相应地更新导航状态：

javascript
watch(
  () => route,
  (newVal, oldVal) => {
    // 检查是否在相关页面
    // 处理 URL 路径参数
    const path = newVal.query.path
    category.value = newVal.params.category
    if (path == undefined) {
      init()
    } else {
      getNavigationFolder(path)
      let pathArray = path.split('/')
      currentFolder.value = { fileId: pathArray[pathArray.length - 1] }
      doCallback()
    }
  },
  { immediate: true, deep: true },
)

// 定义一个函数，用于执行回调
const doCallback = () => {
  // 触发navChange事件，传递categoryId和curFolder的值
  emit('navChange', {
    categoryId: category.value,
    curFolder: currentFolder.value,
  })
}

// Mian.vue中
// 导航改变回调
//解构赋值，从data中获取categoryId和curFolder
const navChange = (data) => {
  const { categoryId, curFolder } = data
  currentFolder.value = curFolder
  category.value = categoryId
  loadDataList()
}

const getNavigationFolder = async (path) => {
  let url = api.getFolderInfo
  if (props.shareId) url = api.getFolderInfo4Share
  if (props.adminShow) url = api.getFolderInfo4Admin

  let result = await proxy.Request({
    url: url,
    showLoading: false,
    params: {
      path: path,
      shareId: props.shareId,
    },
  })

  if (!result) return
  folderList.value = result.data
}

核心导航组件详解 ​

Navigation.vue 组件 ​

组件架构 ​

1、Navigation.vue是文件导航系统的核心组件，负责面包屑导航显示和文件夹层次管理。

2、组件维护两个关键状态变量：

  • folderList: 存储面包屑路径中的文件夹对象数组
  • currentFolder: 当前活动文件夹对象，包含fileId和fileName属性

3、导航界面采用面包屑设计，提供直观的层次化浏览体验：

界面渲染逻辑包含多个导航控制元素：

  • 返回上一级按钮: 仅在有上级目录时显示，点击调用backParent()函数
  • 全部文件链接: 始终可见，点击返回根目录
  • 文件夹路径链: 动态渲染面包屑路径，除最后一个文件夹外都可点击
  • 当前文件夹: 路径末端的文件夹名称，以文本形式显示

标准化事件接口 ​

导航系统通过标准化的事件接口与父组件通信：

navChange事件携带两个关键参数：

  • categoryId: 当前文件分类过滤器
  • curFolder: 当前文件夹对象，包含fileId和fileName

文件夹操作函数 ​

导航组件提供多个核心操作函数：

openFolder函数处理文件夹打开操作：

该函数将新文件夹添加到面包屑路径，更新当前文件夹状态，并同步 URL 路径。

setCurrentFolder函数处理面包屑导航点击：

支持两种操作模式：

  • index = -1: 返回根目录，清空文件夹列表
  • index >= 0: 导航到指定层级，截断后续路径

API 集成与多模式支持 ​

导航系统支持三种不同的操作模式，通过条件 API 选择机制实现：

API 选择逻辑在getNavigationFolder函数中实现：

模式触发条件API 端点参数
分享模式props.shareId存在/showShare/getFolderInfopath, shareId
管理员模式props.adminShow = true/admin/getFolderInfopath
常规模式默认情况/file/getFolderInfopath

路由集成与 URL 同步 ​

导航系统与 Vue Router 深度集成，实现浏览器 URL 与文件夹路径的双向同步：

路由监听器实现了智能的路径过滤：

  • 只有当路径包含/main、/settings/fileList或/share时才会触发导航更新，确保导航逻辑仅在相关页面生效。

  • 路径同步机制通过setPath()函数实现：

  • 该函数将当前文件夹路径转换为 URL 查询参数，实现浏览器前进后退功能和可书签化的文件夹 URL。

文件夹选择对话框系统 ​

FolderSelect.vue 组件设计 ​

FolderSelect.vue组件提供模态对话框界面，用于文件移动操作中的目标文件夹选择：

对话框配置定义了用户界面元素：

包含标题"移动到"和主要操作按钮"移动到此"，点击后触发文件夹选择确认流程。

文件夹数据加载机制 ​

文件夹选择对话框通过loadAllFolder函数动态加载可选择的文件夹：

该函数调用/file/loadAllFolder API，传递当前父文件夹 ID 和过滤条件，获取当前目录下的所有子文件夹。
